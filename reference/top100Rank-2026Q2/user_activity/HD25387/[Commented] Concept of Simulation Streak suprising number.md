### 评论 #6 (作者: SR82953, 时间: 1年前)

I think you might be confusing simulation streak with the number of simulations done. To clarify, simulation streak doesn’t count how many simulations or alpha submissions a consultant has done—it simply reflects how many  **consecutive days**  a consultant has run at least one simulation on the platform. With each passing day of consistent activity, the streak increases by 1. If a consultant misses even one day, the streak resets to zero. Also, unlike other tie-breaker metrics, this one  **doesn’t reset at the start of a new quarter** , which is why you can see very high values. So, maintaining daily consistency is key if you want to build this metric in the Genius program.
