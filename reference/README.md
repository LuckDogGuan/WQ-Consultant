# WorldQuant Brain 顾问系统 - 参考资料与扩展指引 (reference/)

本目录（`reference/`）是项目的核心知识库与扩展参考中心。设立本目录的宗旨在于：汇总 WorldQuant Brain 官方政策白皮书、社区优秀的技术研究成果以及实用的批处理脚本，为系统的后续功能扩展、算法升级提供理论基础与代码参考。

---

## 📂 目录结构与分类说明

目前，本目录共分为以下四个核心子目录：

### 1. `verified/` (已验证的算法与工具)
* **定位**：经评估且测试通过的社区优质算法与逻辑。
* **特性**：此目录下的代码逻辑（如自相关剪枝、正则语法解析器等）与项目目前的底层架构（SQLite 数据库任务流、FastAPI 后端、一/二阶因子生产线）天然兼容。
* **使用建议**：可直接在 `consultant_core/` 或 `app/services/` 中引用或进行重构集成，作为项目功能升级的首选算法来源。
* **现有内容**：
  * `结合自相关做剪枝，提高回测效率代码优化.md` (Article ID: 34140760112535)：一阶因子间的互相关度贪心剪枝逻辑。
  * `构建因子表达式解析器对数据进行高效剪枝-prune 函数新方法实现代码优化.md` (Article ID: 30584578183063)：用于多阶因子表达式提取数据字段和算子特征的结构解析器 `RegularBuilder`。

### 2. `unverified/` (未验证或纯经验性资料)
* **定位**：定性的经验分享、不具备直接执行环境的代码，或与目前系统技术架构不契合的独立脚本。
* **特性**：例如仅限 SuperAlpha 的脚本（当前项目仅支持 Regular 因子流）、针对特定年份的硬编码数据抓取脚本、或者纯文字经验贴。
* **使用建议**：仅作为顾问日常开发、点塔或六维指标提升的阅读参考，**禁止**在未经过大规模重构前直接合入系统代码中。
* **现有内容**：
  * `【新人向-SA的本地计算SC】复制即用！跟上后一篇PC检测代码！代码优化.md` (Article ID: 38081199177239)：针对 SuperAlpha 的本地相关性 Excel 导出脚本。
  * `VF0.98+Expert直升Grandmaster我是怎么做的.md` (Article ID: 37555582592023)：顾问晋升 Grandmaster 的综合策略分享。

### 3. `code/` (第三方与社区辅助脚本)
* **定位**：独立的自动化批量运行脚本与工具（通常需要单独的命令行调用或后台 Job 调度）。
* **使用建议**：用于在项目主体系统之外，辅助进行大批量数据清洗、特定主题（如 PPA 预测能力 Alpha）的数据集或字段导出。
* **现有内容**：
  * `fetch_ppa_datasets.py`：PPA（Predictive Power Alpha）主题数据集及字段的批量获取及汇总导出工具。

### 4. `official_paper/` (官方标准与规范文档)
* **定位**：WorldQuant Brain 官方发布的规范性 PDF 或 Markdown 政策指南。
* **使用建议**：用于校对因子是否合规、检查评审等级指标变化，为系统在计算 `Genius`、`Sharpe`、`Fitness` 和等级达标进度（如 Grandmaster 资格检查）时提供官方判定依据。
* **现有内容**：
  * `Eligibility_Criteria_Grandmaster.md`：官方关于 Grandmaster (GM) 等级评审细则的规范文档。

## 🚀 后续扩展指引 (For Future Extensions)

当顾问或系统后续需要探索新功能时，请遵循以下流程来利用和扩展本目录：

1. **信息收集与抓取**
   * 使用 `wq mcp` 中集成的 `read_forum_post` 或 `search_forum_posts` 工具，检索社区最新的优秀方案，以 Article ID 或 URL 形式将内容下载下来，转化为结构化的 Markdown 文件。
2. **可行性评估与分类**
   * **评估点 1：技术兼容性**。该方案的代码是否使用了与现有项目严重冲突的机制（例如现有的 SQLite/FastAPI 架构 vs 纯 Excel 报表流）？
   * **评估点 2：业务实用性**。它针对的是 Regular Alpha 还是 Super Alpha？它是否包含具体的、可复现的算子/因子逻辑？
   * 评估完成后，将通过验证的代码级文献归入 `verified/`，将经验分享或暂时无法集成的独立脚本归入 `unverified/`。
3. **集成与重构**
   * 针对 `verified/` 下的成熟理论，在 `tests/` 目录下编写验证性单元测试（Modularity & Tests），确保该算法与现有因子流结合时不会引发未定义引用或逻辑冲突，然后再正式重构入主程序。
