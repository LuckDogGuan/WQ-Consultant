# WorldQuant Consultant GUI Progress Report (GUI Enhancements)

本篇文档记录了对 WorldQuant Consultant GUI 交互体验、样式及功能做出的全面增强进度。

## 完成进度与特性清单

### 1. 样式与视觉修复
* **下拉框白色背景修复**：修复了在 Windows 暗黑主题下 select option 元素渲染为白色背景的体验 bug，确保在所有现代浏览器中下拉框均采用深色背景 (`#0f172a`)，文字为白色 (`--text-main`)。
* **浮动 Toast 提示组件**：移除了保存配置时全页刷新导致的位置跳动，增加了屏幕右上角平滑滑入滑出的 Toast 通知，支持 success 与 error 两种视觉类型。

### 2. 设置页面交互优化
* **时间窗输入优化**：将原本需要文字输入的中国时间禁用开始/结束控件改为原生的 `<input type="time">` 控件，提供更直观的时钟选择界面并确保格式正确。
* **密码显隐切换**：自动将已保存的 WorldQuant 密码渲染在密码输入框中。在右侧嵌套了眼睛图标 (`fa-eye` / `fa-eye-slash`)，点击即可一键在明文和密文星号之间切换，极大方便了密码的查看与修改。
* **回测地区与股票池级联联动**：定义了 Region 到 Universe / Delay 的映射联动。当切换 Region 时，Universe 和 Delay 选项框会自动根据支持的列表动态更新，杜绝选择 invalid 范围导致的后台运行失败。
* **AJAX 异步保存**：配置提交事件已被 AJAX fetch 拦截，保存时不发生全页刷新和滚动位置重置，并通过 Toast 提供实时成功/失败状态。

### 3. 数据目录 Scope 跨页面切换与刷新
* **跨地区直接查看**：在 `/catalog` 页面顶部直接植入了 Region、Universe、Delay 的下拉级联选择框，点击“切换查看该区域”按钮即可立即加载并重载查看所选 Scope 下的缓存数据集，无需再前往设置页面频繁更改默认设置。
* **未缓存提示**：在下拉框下方增加了未缓存提示条。当所选 Scope 在本地不存在缓存时，会触发警示提示“目标地区未缓存”。
* **一键刷新整区数据**：在 `/catalog` 页面提供了“刷新同步该地区数据”按钮，点击后直接发起 AJAX 任务启动后台的 `catalog_refresh` 刷新任务，同步刷新该地区全部数据集的目录缓存，随后自动引导至 `/dashboard` 查看进度。

### 4. 任务参数显示与进度个数显示
* **`/api/jobs` 接口扩展**：接口返回数据中新增了 `params` 对象，以向前端提供各个任务的执行参数。
* **任务参数子展示**：仪表盘、回测、相关性分析、提交检查的任务表格行中，任务标题下方新增了微型图标及参数子项展示。能直观看出如“一阶: 是 | 二阶: 否 | 数据集: macro38”等详细入参。
* **个数进度展示**：
  - 各个服务（`simulation_service.py` 跑 pools、`check_service.py` 跑 check、`correlation_service.py` 跑 check alphas）中进度更新逻辑全面更新，`progress_current` 存储已执行的物理个数，`progress_total` 存储队列总物理个数。
  - 前端 `app.js` 与 Jinja2 初次渲染代码同步重构，在进度条内展示类似 `12 / 56 (21%)` 格式的详细进度，更具确定感。
* **前端更新代码健壮性重构**：将 `app.js` 原本依赖列索引（`row.cells[i]`）更新单元格的逻辑改为通过类选择器（如 `.job-progress-cell`）精确更新，彻底规避了因不同页面任务表格列数不一致导致的界面渲染错乱。

### 5. 地区-股票池级联关系的动态优化 [New Feature]
* **读取 day1_scope_index 对应关系**：摒弃了硬编码的静态映射，后台通过解析 `day1_scope_index.xlsx` 提取该工作区可用的地区、股票池和延迟（例如 CHN 仅对应 `TOP2000U`，USA 对应七类股票池等）。
* **动态下拉框呈现**：在设置页面（`/settings`）及数据目录页面（`/catalog`），地区下拉列表和关联的股票池、延时下拉列表均基于 `day1_scope_index.xlsx` 真实可用数据进行级联填充与渲染，确保前端下拉选项与 WorldQuant Brain 可用目录完全保持同步，提高了可靠性。

### 6. 修改属性二次确认与任务全面操作 [Enhanced]
* **修改属性二次确认**：在系统设置页面新增了“交互与安全设置”选项卡，允许用户控制修改配置属性或任务参数时是否需要二次点击确认，默认关闭（即默认不需要二次确认）。开启后，无论是保存系统设置表单还是修改任务参数，前端都会进行 `confirm` 二次弹窗拦截。
* **任务的“启动、停止、修改、删除”四项标准操作**：
  - **启动/继续**：允许对已暂停、失败或完成的任务进行一键重启/继续。
  - **停止**：对运行中的任务提供“停止”操作（原暂停），可通知后台工作线程安全退出。
  - **修改**：提供精美的任务修改 Modal 弹窗。针对不同任务类别（如批量回测、数据目录刷新等）自适应渲染可配置参数，允许用户直接进行参数修改。
  - **删除**：提供“删除”按钮，安全地将任务及其关联事件从数据库中清空，如果任务在运行中则会同步发出停止信号，并实时移除前端表格的对应行。
* **相关性诊断自动改名勾选框**：在“相关性检测”页面顶部启动区域新增了“诊断合格后自动在平台改名 (Auto Rename)”勾选框，默认处于勾选状态。允许用户在创建相关性诊断任务时，决定是否对诊断出的 PPA/ATOM/RA 优质候选自动进行平台远程改名。
* **回测 NameError 异常修复**：排查并修复了在 `simulation_service.py` 内部 `run_backtest_job` 函数在执行二阶 (SO) 或三阶 (TH) 阶段时，由于凭证变量（`username`, `password`）未在函数主体中定义，导致调用 `login_with_credentials` 时抛出 `NameError: name 'username' is not defined` 的致命 Bug。
* **SQLite 约束冲突异常修复**：修复了在 `upsert_alpha` 存储 Alpha 信息时，部分字段（如 `name`）由于平台数据返回为 `None`，导致 SQLite 抛出 `NOT NULL constraint failed: alpha_records.name` 写入失败 of Bug。已通过空值合并逻辑（`or ""`）确保强制合并为字符串。
* **网络断开监测与任务自动挂起/恢复 [New Feature]**：
  - **后台监视线程**：新设了后台网络状态监视器 `NetworkMonitor` 线程，每 10 秒对 `https://api.worldquantbrain.com` 连通性进行一次轻量化扫描检测。
  - **网络断开时**：自动将所有处于活动中（`running`、`reconnecting`、`waiting_limit`、`waiting_time_window`）的后台任务标记并挂起为 `waiting_network` 状态，输出断开日志与事件，并终止当前执行线程。
  - **网络恢复时**：若监视器检测到连通恢复，会自动寻找所有处于 `waiting_network` 状态的挂起任务，将其重新拉起恢复运行，实现彻底的“自动断线恢复”。
  - **前端顶栏状态联动**：前端通过 AJAX 轮询该状态，对页面顶部的“系统状态指示灯”进行秒级联动更新。断开时指示灯立即变红，并显示“网络连接已断开 (10秒轮询中...)”；恢复时即刻重归绿色常亮，提示“系统状态正常”。
